# AWS Shared Responsibility and Artifact Reference

## Shared Responsibility Summary

AWS manages security *of* the cloud (physical infrastructure, foundational services).
Customers manage security *in* the cloud (configuration, identities, data, workload controls).

## How This Platform Uses AWS Artifact

AWS Artifact is used to obtain:

- AWS compliance reports (e.g., ISO, SOC)
- Certifications and attestations supporting supplier assurance
- Audit support documentation for inherited controls

## Usage Guidance

- Track which controls are inherited from AWS vs customer-managed.
- Store references to retrieved reports in the audit evidence process.
- Revalidate report currency during each audit cycle.
